# Weekly Narrative Report — Week 3 (May 18 to May 22)

Project Overview

In the third weekly interval the project will focus on consolidating prior integration work into a demonstrable deliverable: robust telemetry streams, reliable command control, and complete operational documentation. The scope for May 18 to May 22 targets full-system integration testing, field trials in a controlled environment, and the finalization of the user manual and pin mapping artifacts.


Weekly Activities

Planned activities include executing end-to-end field tests that exercise the ESP32, the Arduino sensor hub, motor drivers, and the Android control application in realistic scenarios. The team will run extended endurance tests to validate thermal and power characteristics, perform sensor validation across multiple temperature points, and iterate on user flows in the mobile app based on operator feedback. Documentation activities will consolidate observed wiring changes, firmware configuration options, and operational checklists into the `UsersOperational Manual Guide` and associated README updates.

Challenges Anticipated

Anticipated issues include environmental variability during field trials that may expose new sensor calibration needs, potential RF interference impacting Wi‑Fi connectivity, and the need to refine power distribution under sustained motor loads. Contingency plans include portable spectrum checks, temporary tethered Ethernet for controlled tests if Wi‑Fi proves unreliable, and providing backup power sources for repeatable endurance testing.

Learnings & Insights

The third week is expected to further strengthen best practices around system validation: documenting reproducible test procedures, capturing telemetry to support post-test analysis, and formalizing firmware update procedures. The team also plans to refine the user manual writing process, using clear stepwise procedures and annotated pin diagrams to reduce setup errors for future users.

Next Steps

After completing the field tests and finalizing documentation, the immediate next steps are to consolidate release artifacts (firmware binaries, Android APK, and documentation bundle), perform a staged handover of the operational guide, and prepare a short demonstration to review project readiness with stakeholders.

Conclusion

Week 3 is organized to validate and close the integration loop: verify system reliability under realistic conditions, capture findings in documentation, and prepare deliverables for demonstration. If field tests proceed as planned, the project will be ready for an initial stakeholder demonstration and further refinement cycles driven by user feedback.
